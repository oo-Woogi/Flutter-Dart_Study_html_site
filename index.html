3. Dart에서 dynamic 키워드로 선언한 변수에 대한 설명으로 옳은 것은?

a) dynamic으로 선언된 변수는 한번 값이 할당되면 타입이 고정되어 변경할 수 없다
b) dynamic 변수는 반드시 모든 연산이 컴파일 시 타입 검사를 통과해야만 실행된다
c) dynamic 변수는 object와 동일하여, 메서드 호출 시에도 항상 컴파일러가 존재 여부를 검사한다
d) dynamic 변수는 타입이 고정되지 않아 실행 중에 서로 다른 타입의 값을 저장할 수 있다

정답: d) dynamci 변수는 타입이 고정되지 않아 실행 중에 서로 다른 타입의 값을 저장할 수 있다

💡

a)는 final 키워드(재할당 불가)나 var 최초 추론 고정 개념을 섞은 잘못된 설명

b)는 dynamic은 정적(compile-time) 타입검사가 약화되어 런타임 오류 가능성↑

c)는 object는 선언된 멤버만 호출 가능, dynamic은 임시로 "모든 멤버 있는 것처럼" 취급

d)는 사용할 수 없는 주석입니다.

4. Dart에서 final과 const의 차이에 대한 설명으로 옳은 것은?

a) final 변수는 컴파일 타임에 반드시 값이 확정되어야 하고, const 변수는 런타임 첫 대입 때 결정된다

c) final 변수는 재할당은 가능하지만 내부 필드 수정을 막고, const 변수는 재할당이 가능하다
d) final 과 const 는 완전히 동일하며 성능·메모리 측면에서도 차이가 없다

정답: b) final 은 “한 번만 할당(런타임에 결정 가능)”, const 는 “컴파일 타임 상수(값이 미리 확정되고 불변 객체)”를 의미한다

a)는 두 키워드 역할을 바꿔 적음

35. List&lt;int&gt; numbers = [1, 2, 3];가 있을 때, map() 함수를 사용하여 각 요소를 제곱한 새로운 List를 만드는 코드를 작성하세요.

💡 map() 함수는 리스트의 각 요소를 순회하며 주어진 함수를 적용하여 새로운 Iterable을 반환합니다.

36. List&lt;int&gt; numbers = [1, 2, 3, 4, 5];가 있을 때, reduce() 함수를 사용하여 모든 요소의 합을 구하는 코드를 작성하세요.

정답: int sum = numbers.reduce((value, element) =&gt; value + element);

💡 reduce는 리스트의 첫 번째 요소를 초기값으로 사용하고, 두 번째 요소부터 순회하며 누적 연산을 수행합니다.

37. List&lt;int&gt; numbers = [1, 2, 3];가 있을 때, fold() 함수와 초기값 10을 사용하여 모든 요소의 합을 구하는 코드를 작성하세요.

정답: int sum = numbers.fold(10, (previousValue, element) =&gt; previousValue + element);

💡 fold는 reduce와 유사하지만, 연산을 시작할 초기값을 직접 지정할 수 있습니다. (결과: 10 + 1 + 2 + 3 = 16)

38. reduce()와 fold()의 가장 큰 차이점에 해당하는 것을 모두 고르세요.

a) reduce는 숫자 리스트에만 사용할 수 있다.
b) fold는 초기값을 지정할 수 있다.
c) reduce는 비어 있는 리스트에서 사용할 수 없다.
d) fold는 반환 타입이 항상 int이다.

정답: b), c)

💡 any() 메서드는 리스트의 요소 중 하나라도 주어진 조건을 만족하면 true를 반환하고, 아니면 false를 반환합니다.

Part 4: 단어 정의 (Glossary)

Dart & Flutter 학습 중 자주 등장하는 핵심 용어를 한 곳에 정리했습니다.

💡 Immutable (불변)

생성 후 내부 상태(값)가 변경되지 않는 객체. const 객체는 불변이며 동일 값 재사용(canonicalization) 가능.

💡 Null Safety

컴파일 시점에 null 가능성을 추적하여 런타임 null 오류를 줄이는 Dart의 타입 시스템 기능. T?는 null 허용, T는 불허.

💡 Widget

Flutter UI를 구성하는 최소 단위. UI + 상태 + 레이아웃 정보의 선언적 표현.

💡 State (상태)

위젯이 현재 렌더링을 결정하는 데이터. 변경 시 UI를 다시 빌드하도록 트리거.

💡 Build Context

위젯 트리 내 위젯의 위치·상위 InheritedWidget 접근을 위한 핸들.

💡 var (타입 추론)

변수를 선언할 때 사용하는 키워드. 할당되는 값을 기준으로 컴파일러가 타입을 자동으로 추론하며, 한 번 타입이 정해지면 변경되지 않음.

💡 final (런타임 상수)

단 한 번만 값을 할당할 수 있는 변수를 선언. 런타임 시점에 값이 결정되며, 한 번 할당된 후에는 변경 불가.

💡 const (컴파일 타임 상수)

컴파일 시점에 값을 확정하는 상수를 선언. final보다 엄격하며, 객체의 내용까지도 완전히 불변으로 만듦.

💡 dynamic (동적 타입)

모든 타입을 허용하는 변수. 컴파일 시점에는 타입 검사를 하지 않고, 런타임에 타입이 결정되어 유연하지만 타입 안정성은 낮음.

💡 late (지연 초기화)

Null-safe 환경에서 non-nullable 변수의 초기화를 나중에 할 것임을 명시. 변수에 접근하기 직전에 반드시 초기화되어야 함.

💡 Null Safety (널 안정성)

코드에서 null 때문에 발생하는 오류를 방지하기 위한 Dart의 타입 시스템 기능. 변수는 기본적으로 null을 허용하지 않으며(non-nullable), ?를 붙여야만 null을 허용함.

💡 Collection (컬렉션)

여러 값을 하나의 단위로 묶어서 관리하는 객체. Dart의 대표적인 컬렉션으로는 List, Map, Set이 있음.

💡 Iterable (이터러블)

for-in 루프를 통해 요소를 순차적으로 접근할 수 있는 객체들의 추상적인 묶음. List와 Set이 대표적인 Iterable이며, where, map 등의 메서드는 Iterable을 반환함.

💡 Collection (컬렉션)

여러 값을 하나의 단위로 묶어서 관리하는 객체. Dart의 대표적인 컬렉션으로는 List, Map, Set이 있음.

💡 Iterable (이터러블)

for-in 루프를 통해 요소를 순차적으로 접근할 수 있는 객체들의 추상적인 묶음. List와 Set이 대표적인 Iterable이며, where, map 등의 메서드는 Iterable을 반환함.

💡 List (리스트)

순서가 있는 데이터의 묶음. 인덱스(0부터 시작)를 통해 각 요소에 접근할 수 있으며, 중복된 값을 허용함.

💡 Map (맵)

Key와 Value를 하나의 쌍으로 묶어 저장하는 컬렉션. Key는 고유해야 하며, 해당 Key를 통해 Value에 빠르게 접근할 수 있음.

💡 Set (셋)

순서가 없고, 중복된 요소를 허용하지 않는 컬렉션. 특정 요소의 존재 여부를 빠르게 확인할 때 유용함.

💡 enum (열거형)

서로 연관된 몇 개의 상수 값을 하나의 타입으로 묶어서 정의하는 기능. 코드의 가독성과 안정성을 높여줌 (예: 상태, 색상 등).

💡 Null-aware Operators (널 인식 연산자)

null 값을 안전하게 처리하기 위한 연산자.
- ?. (Null-aware access): 객체가 null이 아니면 속성에 접근, null이면 null 반환.
- ?? (If-null): 왼쪽 값이 null이면 오른쪽 값을 사용.
- ??= (Null-aware assignment): 변수가 null일 때만 값을 할당.

💡 async / await

비동기 코드를 동기 코드처럼 간결하게 작성하게 해주는 키워드. Future나 Stream과 같은 비동기 작업을 처리할 때 사용됨. async는 함수가 비동기임을, await는 작업이 끝날 때까지 기다림을 의미.

💡 Future (퓨처)

미래의 특정 시점에 완료될 하나의 비동기 작업의 결과를 나타내는 객체. 성공적인 값 또는 오류를 담을 수 있음.

💡 Lambda / Anonymous Function (람다 / 익명 함수)

이름이 없는 함수. 다른 함수의 인자로 전달되거나, 컬렉션의 forEach, map 등에서 간결한 로직을 구현할 때 주로 사용됨 (예: (n) => n * 2).

💡 typedef (타입 별칭)

복잡한 타입(주로 함수 타입)에 새로운 이름을 붙이는 기능. 코드의 가독성을 높이고 재사용성을 좋게 만듦.

💡 try-catch

프로그램 실행 중 발생할 수 있는 오류(예외)를 처리하는 구문. try 블록에서 오류를 감시하고, catch 블록에서 발생한 오류를 잡아 처리함.

💡 Spread Operator (...) (스프레드 연산자)

하나의 컬렉션 안에 다른 컬렉션의 모든 요소를 펼쳐서 넣는 기능. 리스트나 맵을 합칠 때 유용함.

💡 Cascade Notation (..) (캐스케이드 표기법)

동일한 객체에 대해 여러 메서드 호출이나 속성 변경을 연속적으로 수행할 수 있게 하는 구문. 코드를 더 간결하게 만들어 줌.

💡 Positional Parameters (위치 기반 파라미터)

함수를 호출할 때 정의된 순서대로 값을 전달해야 하는 파라미터. 가장 기본적인 함수 파라미터 방식.

💡 Named Parameters (이름 기반 파라미터)

{}로 감싸서 정의하며, 함수 호출 시 이름을 명시하여 값을 전달하는 파라미터. 순서와 상관없이 값을 전달할 수 있어 가독성이 높음.

💡 required (필수 파라미터)

네임드 파라미터와 함께 사용하여, 해당 파라미터가 선택이 아닌 필수로 값을 받아야 함을 명시하는 키워드.

💡 Type Checking Operators (타입 확인 연산자)

객체의 타입을 확인할 때 사용하는 연산자.
- is: 객체가 특정 타입이면 true 반환.
- is!: 객체가 특정 타입이 아니면 true 반환.

💡 Integer Division Operator (~/) (정수 나누기 연산자)

나눗셈을 수행한 후, 결과값의 소수점 이하를 버리고 정수 몫만 반환하는 연산자.

💡 Pattern Matching (패턴 매칭)

switch문이나 if문 등에서 값의 구조나 타입을 분해하고 검사하는 기능. 복잡한 조건 분기를 훨씬 간결하고 강력하게 만들어 줌.

💡 when Clause (가드 절)

패턴 매칭에서 case에 추가적인 조건을 부여할 때 사용하는 구문. 패턴이 일치하더라도 when 조건이 true여야 해당 코드가 실행됨.

💡 Sealed Class (봉인 클래스)

미리 정해진 서브 클래스 외에는 상속을 막는 클래스. switch문에서 Sealed Class의 모든 서브 타입을 검사하도록 강제할 수 있어, 상태 관리에 유용함.

💡 identical() (동일성 확인 함수)

두 변수가 메모리상에서 완전히 동일한 객체를 참조하는지 확인하는 함수. ==가 값의 동등성을 비교하는 것과 달리, identical()은 참조의 동일성을 비교함.

💡 tryParse() (안전한 파싱)

문자열을 다른 타입(예: int)으로 변환할 때 사용하는 안전한 메서드. 변환에 실패하면 오류를 발생시키는 parse()와 달리, null을 반환하여 프로그램이 중단되는 것을 막아줌.

💡 Ternary Operator (? :) (삼항 연산자)

조건 ? 참일_때_값 : 거짓일_때_값 형태로, 간단한 if-else 문을 한 줄로 표현할 수 있는 연산자.

💡 Event Loop (이벤트 루프)

Dart가 비동기 코드를 처리하는 핵심 메커니즘. Microtask Queue와 Event Queue라는 두 개의 큐를 사용하여 작업을 관리하며, Microtask 큐가 항상 우선적으로 처리됨.

💡 compareTo() (비교 메서드)

두 객체를 비교하여 순서를 결정하는 메서드. sort()와 같은 정렬 함수 내부에서 사용되며, 음수, 0, 양수를 반환하여 순서를 정함.

💡 removeWhere() (조건부 삭제)

컬렉션(주로 List)에서 주어진 조건을 만족하는 모든 요소를 제거하는 메서드.

💡 any() (하나라도 만족)

컬렉션의 요소 중 하나라도 주어진 조건을 만족하면 true를 반환하는 메서드.

💡 every() (모두 만족)

컬렉션의 모든 요소가 주어진 조건을 만족해야만 true를 반환하는 메서드.

💡 expand() (펼치기)

중첩된 컬렉션의 각 요소를 꺼내어 하나의 1차원 컬렉션으로 펼쳐주는 메서드.

💡 firstWhere() (조건에 맞는 첫 요소 찾기)

컬렉션에서 주어진 조건을 만족하는 첫 번째 요소를 찾는 메서드. orElse를 통해 조건에 맞는 요소가 없을 경우의 대체 동작을 지정할 수 있음.

💡 Map.fromEntries() (항목으로 맵 생성)

MapEntry(키-값 쌍)의 리스트를 받아 새로운 Map을 생성하는 생성자.

💡 Top-level Function (최상위 함수)

클래스나 다른 함수에 속하지 않고, 파일의 가장 바깥 수준에 선언된 함수. main() 함수가 대표적인 예.

💡 .toList() / .toSet()

where, map 등 Iterable을 반환하는 메서드의 결과를 실제 List나 Set 컬렉션으로 변환하는 메서드. 이 과정을 거쳐야 리스트의 인덱스 접근([]) 같은 고유 기능을 사용할 수 있음.

💡 Method Chaining (메서드 체이닝)

하나의 객체에 여러 메서드를 .으로 연결하여 연속적으로 호출하는 기법 (예: numbers.where(...).map(...).toList()). 코드를 간결하고 흐름에 따라 읽기 쉽게 만들어 줌.

💡 runtimeType (런타임 타입)

모든 객체가 가지고 있는 속성으로, 프로그램 실행 중에 해당 객체의 실제 타입을 반환함. 디버깅이나 동적 상황에서 타입을 확인할 때 유용함.

💡 Fall-through (폴스루)

switch문에서 case 블록의 끝에 break가 없을 경우, 다음 case 블록으로 실행이 계속 이어지는 현상. 의도적으로 사용될 수 있으나 버그의 원인이 되기도 함.

💡 main() Function (메인 함수)

모든 Dart 프로그램의 시작점(Entry Point). 프로그램이 실행될 때 가장 먼저 호출되는 최상위 함수.

💡 void Return Type (void 반환 타입)

함수가 아무런 값도 반환하지 않음을 명시하는 타입. 값을 return하지 않는 함수의 반환 타입으로 사용됨.

💡 on Keyword (예외 타입 지정)

try-catch 구문에서 catch 대신 특정 타입의 예외만 잡고 싶을 때 사용하는 키워드 (예: on FormatException { ... }). 코드의 안정성을 높여줌.

💡 Future.delayed

지정된 시간만큼 기다린 후에 특정 작업을 수행하는 Future를 생성하는 생성자. 비동기적으로 지연이 필요할 때 사용됨.

💡 Future.microtask

이벤트 루프의 마이크로태스크 큐에 작업을 예약하는 Future 생성자. 일반 이벤트(Future(...))보다 항상 먼저 실행됨.

💡 MapEntry (맵 엔트리)

맵(Map)에 포함되는 단일 키-값 쌍을 나타내는 객체. map.entries를 통해 Map의 모든 항목을 MapEntry의 Iterable로 얻을 수 있음.
